# Concept 2 — Momentum Mesh

**Tagline:** Personal GTD, powered by social momentum.

## Primary Differentiator
Accountability circles that make GTD execution collaborative without turning the app into full project management software.

## 3 Key Differentiators
1. **Accountability pods** (2–5 people) with optional weekly commitments.
2. **“Waiting For” handshake** where delegated items are explicitly accepted/declined.
3. **Momentum feed** shows completed next actions, not noisy chat.

## Target User
Freelancers, creators, and small founder teams who want gentle accountability and follow-through on commitments.

## Core Features (max 5)
1. Shared commitment board (weekly top 3 outcomes).
2. Delegation + acknowledgment flow for “Waiting For”.
3. Check-in ritual templates (weekly review prompts).
4. Private mode for sensitive projects, selective sharing controls.
5. Progress nudges based on upcoming commitments.

## Tech Stack Recommendation
- **Frontend:** Flutter (mobile-first) + lightweight web client
- **Backend:** Supabase/Postgres + Realtime channels
- **Auth:** OAuth + passkeys
- **Analytics:** Event pipeline for streak and commitment completion metrics

## Research Notes (existing GTD pain points addressed)
- Most GTD tools are solitary; users lose consistency without external accountability.
- “Waiting For” lists become stale because there is no recipient acknowledgment.
- Team tools are too heavy for personal productivity contexts.
