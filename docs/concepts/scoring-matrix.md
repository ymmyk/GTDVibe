# GTD MVP Concepts — Comparative Scoring Matrix

## Methodology (Velen)
Scored 1–5 (higher is better) across:
- **Feasibility (35%)**: speed/risk to deliver MVP with small team
- **User Value (45%)**: likelihood of solving painful, frequent GTD problems
- **Dev Effort Efficiency (20%)**: implementation complexity relative to expected learning

**Weighted Score Formula:**
`(Feasibility × 0.35) + (User Value × 0.45) + (Dev Effort Efficiency × 0.20)`

> Note: Dev Effort Efficiency is inverse to raw effort (high score = less painful build for value gained).

## Rationale Snapshot
- **FlowCapture** wins Wave 0 because capture friction is universal and MVP scope is controlled.
- **Context Compass** has high upside; recommendation quality is a strong retention driver.
- **Loop Ledger** is operationally valuable and differentiates on trust/review discipline.
- **Momentum Mesh** is compelling for accountability cohorts, but adoption depends on social behavior.
- **Ops GTD** offers strong strategic differentiation, but graph/dependency UX raises MVP complexity.
